# E3 — Package Sekai as a double-clickable, unsigned macOS Electron app (electron-builder): built dist served by a bundled local Node server child, no terminal, no npm start.

Turns Sekai from a "run npm and open a browser" dev toy into a self-contained Sekai.app you double-click. The core problem: the /api/sessions, /api/events and /api/resume endpoints today live ONLY inside the Vite dev-server middleware (vite.config.js), so a packaged build that serves static files would have a dead API. E3 introduces a standalone zero-dependency Node HTTP server (server/server.js) that serves the vite-built dist/ AND those three endpoints from one 127.0.0.1 origin, reusing the exact same scan.js/gitinfo.js/resume.js logic via a shared router (server/api.js) so dev and prod behave identically. The Electron main process spawns that server as a child on app-ready (Electron binary run as Node via ELECTRON_RUN_AS_NODE), waits for it to print its kernel-assigned port, loads http://127.0.0.1:PORT, and kills the child cleanly on quit. electron-builder produces an arm64 .app (dir) + .dmg, unsigned, with a procedurally generated planet icon. No src/ file and no world-state code is touched.

## New files

### `server/api.js`
Single source of truth for the three API routes, shared by the Vite dev middleware AND the packaged standalone server so /api/* behaves byte-identically in dev and prod. Zero npm deps.

**Contract:** export async function handleApi(req, res) -> Promise<boolean>. Parses path = (req.url||'').split('?')[0]. Route table: '/api/sessions' -> res 200 application/json JSON.stringify(await scanSessions()); '/api/events' -> res 200 application/json JSON.stringify(await gitEvents()); '/api/resume' -> if req.method==='POST' await handleResume(req,res) else res.statusCode=405,res.end(); anything else -> return false (NOT an api route). On any thrown error inside a handled route: res.statusCode=500, application/json, res.end(JSON.stringify({error:String(e)})). Returns true iff it owned the request (wrote a response), false to let the caller fall through to static serving / vite next(). Imports: import { scanSessions } from './scan.js'; import { gitEvents } from './gitinfo.js'; import { handleResume } from './resume.js'. handleResume already sets its own content-type + status and calls res.end, so do not double-write for /api/resume beyond the 405 branch.

### `server/server.js`
The standalone local web server the packaged app runs as a child process: serves the built dist/ static site plus the /api/* endpoints on 127.0.0.1 at a kernel-assigned port, and hands that port back to the Electron main process over stdout. Zero npm deps (node:http, node:fs, node:path, node:url only).

**Contract:** ESM entry, no exports (a runnable script). Reads env: SEKAI_DIST (absolute path to the dist/ web root; required), SEKAI_HOST (default '127.0.0.1'), SEKAI_PORT (default '0'). Step 1 (PATH fix for Finder-launched GUI apps): process.env.PATH = ['/opt/homebrew/bin','/usr/local/bin','/usr/bin','/bin','/usr/sbin','/sbin', process.env.PATH||''].filter(Boolean).join(':') — without this, gitinfo.js's execFile('git'/'gh') and resume.js's execFile('osascript') may not resolve because a double-clicked .app inherits a minimal PATH. Step 2: import { handleApi } from './api.js'. Step 3: const server = http.createServer(async (req,res) => { try { if (await handleApi(req,res)) return; serveStatic(req,res); } catch(e){ res.statusCode=500; res.end(String(e)); } }). serveStatic: let p = decodeURIComponent((req.url||'/').split('?')[0]); if (p==='/'||p==='') p='/index.html'; const filePath = path.join(distRoot, path.normalize(p)); GUARD: if (!filePath.startsWith(distRoot + path.sep) && filePath!==distRoot) -> 403; fs.stat -> if missing or directory -> 404 (plain text); else set Content-Type from ext via MIME map {'.html':'text/html; charset=utf-8','.js':'text/javascript; charset=utf-8','.mjs':'text/javascript; charset=utf-8','.css':'text/css; charset=utf-8','.json':'application/json; charset=utf-8','.svg':'image/svg+xml','.jpg':'image/jpeg','.jpeg':'image/jpeg','.png':'image/png','.webp':'image/webp','.glb':'model/gltf-binary','.gltf':'model/gltf+json','.bin':'application/octet-stream','.wasm':'application/wasm','.ico':'image/x-icon','.md':'text/markdown; charset=utf-8'} default 'application/octet-stream'; pipe fs.createReadStream(filePath) to res. Step 4: server.listen(Number(SEKAI_PORT), SEKAI_HOST, () => { process.stdout.write('SEKAI_LISTENING ' + server.address().port + '\n') }). Step 5: for (const sig of ['SIGTERM','SIGINT']) process.on(sig, () => server.close(() => process.exit(0))). Bind 127.0.0.1 ONLY — never 0.0.0.0 — the /api/resume + git endpoints must not be reachable off-box.

### `scripts/make-icon.mjs`
Generates build/icon.png (1024x1024), a procedural stylized planet on a deep-space background, with zero npm dependencies — matching the codebase's node-builtins-only ethos. electron-builder converts this PNG into the .icns at package time.

**Contract:** ESM script, no exports. Uses only node:zlib and node:fs. Algorithm (fully deterministic, NO Math.random): const W=1024,H=1024; allocate Buffer rgba = W*H*4. For each pixel (x,y): start with space bg (4,6,12). dx=x-512, dy=y-512, d=hypot(dx,dy), R=400. If d<R: base ocean (30,90,150); add 3-4 landmass blobs as fixed gaussian bumps at hardcoded centers/radii (e.g. (430,470,120),(600,560,140),(520,660,90)) tinted green (70,130,60), blended by exp(-dist^2/(2*r^2)); apply day-light gradient by scaling brightness with (0.55 + 0.45*clamp(( -dx-dy )/(1.4*R)+0.5,0,1)) so top-left is lit; . Atmosphere rim: for d in [R-6, R+16], blend toward (120,180,255) by a smoothstep falloff. Encode PNG manually: 8-byte signature 89 50 4E 47 0D 0A 1A 0A; IHDR chunk (W,H, bitDepth 8, colorType 6=RGBA, 0,0,0); build raw scanline buffer = for each row a leading filter byte 0x00 then that row's RGBA; IDAT = zlib.deflateSync(raw); IEND empty. Each chunk = 4-byte big-endian length + 4-byte type ASCII + data + 4-byte big-endian CRC32(type+data). Implement a small CRC32 (standard 0xEDB88320 table). fs.mkdirSync('build',{recursive:true}); fs.writeFileSync('build/icon.png', concat of signature+chunks). Idempotent: safe to run every build.

## main.js wiring
NONE. This epic adds no THREE module and requires zero changes to src/main.js or any src/ file. It defers ONLY the render loop's data source (the same three relative fetch('/api/...') calls the client already makes) onto a new same-origin http://127.0.0.1:PORT server. The only integration point is the Electron main process (electron/main.cjs) plus the new server bootstrap (server/server.js) — there is no create*/scene.add/update(dt) wiring because nothing enters the THREE scene graph.

## Determinism
This epic modifies only electron/main.cjs, the local server bootstrap (new server/api.js, server/server.js), build config (electron-builder.yml, package.json, .gitignore), the shared router refactor in vite.config.js, and an icon generator (scripts/make-icon.mjs). It touches NO file under src/ and NO world-state code, so every seeded RNG path (rngFromString, hash01, makeNoise3D via src/util.js) and the presentation-only dt clock in update(dt) are byte-for-byte unchanged. No Math.random()/Date.now() is introduced into world state. The two non-seeded values in the design are pure infrastructure that never reach the THREE scene: (1) the TCP port — SEKAI_PORT=0 lets the kernel assign an ephemeral port, read back via server.address().port and passed to the window through a stdout 'SEKAI_LISTENING <port>' sentinel (chosen this way specifically to make port conflicts impossible, not to be random); (2) child-process lifecycle timing. make-icon.mjs is fully deterministic (hardcoded sinusoid/gaussian parameters, no RNG), so the icon is reproducible across builds.

## Covenant
N/A — this is a packaging/infra epic. It introduces no simulation and never reads or writes session structures. The only session I/O is the pre-existing read-only scanners (scan.js reads ~/.claude/projects transcripts; gitinfo.js polls git/gh) and the pre-existing resume launcher (resume.js opens Terminal) — all reused unchanged behind the same three endpoints. Session buildings and their persistent history are untouched by definition, since no src/ or world-state file is modified.
