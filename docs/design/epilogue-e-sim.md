# E-SIM — the world-simulation conflict ladder (raids → wars → sieges), covenant-bound

A staged conflict layer that lets sessions-as-civilization go to war without ever touching session history. RUNG 1 ships standalone: a fully seeded skirmish theater where raiding parties (tmp-dir/orc factions) march on prosperous settlements, defenders muster, a field battle clashes with night torches, and the outcome leaves banners + scorch decals + a herald chronicle line — every mark healing back to the exact pre-raid world within one deterministic epoch. The split is warsim.js (pure, testable data/logic — schedule, geometry, outcomes, covenant clearance) + warrender.js (instanced render — armies, marks, torches), mirroring the shipped civsim/civrender pair. Rungs 2 (git-merge-conflict-driven border skirmishes, route-supply interception, prosperity-weighted targeting, territory shading) and 3 (persistent faction standing, supply economy, sieges, peace treaties on merge) build additively on the same two files.

## New files

### `src/warsim.js`
RUNG 1 pure data/logic layer for the conflict ladder — no THREE render, no materials (Vector3 imported from the 'three' core exactly like civsim.js so it is node-testable). Reduces (settlement snapshot, worldSeed, warClock) to a fully deterministic, replayable set of raids: who raids whom, when, the on-land battlefield/muster/defense geometry (all covenant-clearance-tested), per-unit formation slots + casualty times, the seeded outcome, and the mark positions. warrender.js is a pure projection of this.

**Contract:** export function createWarSim(planet, settlements, seed, opts={}) -> { raids, EPOCH, isRaider(project)->bool, strengthOf(settlement)->number, attractivenessOf(settlement)->number, raidStateAt(raid, warClock, out={})->out, outcomeLine(raid)->string, meta:{raidCount} }. `settlements` is an array of {project,name,race,structures,anchorDir:Vector3,groundR,structureDirs:Vector3[]}. Construction: build a covenant set (all anchorDir + all structureDir), pick raider settlements via isRaider (project path matches /(^|\/)(tmp|temp|\.tmp|var\/folders|T\/)/ OR race==='orc'), pair each raider to its most-attractive non-raider target within MAX_RAID_ANGLE (~0.6 rad) — attractiveness = structures-weighted, deterministic ties via hash01; cap at RAID_CAP (4). Per raid compute (all seeded, all clearance-tested against the covenant set with an expanding-ring search, skip+warn-once if no clear spot): id (`raid:`+raider+`>`+target), epochOffset = idx*(EPOCH/RAID_CAP), sourceDir=raider.anchorDir, targetDir=target.anchorDir, musterDir (offset OUTWARD from target toward source, outside footprint), battlefieldDir (midpoint slerp then clearance-nudged), defenseDir (between target and battlefield), atkCount/defCount (clamp(round(strength*k),MIN,MAX)), attackerSlots/defenderSlots ([{bearing,dist,fallAt}] seeded per unit), outcome ('repelled' if defenderStrength*(0.6+0.8*hash01(id+':outcome'))>=attackerStrength else 'raided'), scorchDirs (ring at battlefield + if 'raided' a ring offset AROUND target), bannerDirs ([{dir,forward}] winner-side), propDirs. EPOCH (~90s) sim-seconds. raidStateAt(raid,clock,out): let tau=(clock-raid.epochOffset) wrapped into [0,EPOCH); out.phase in {dormant,marching,mustering,clashing,aftermath,healed}; out.marchT/clashT/aftermathT in 0..1; out.healFrac (0..1 over HEAL_DUR from scorch spawn at clash end); out.atkAlive/defAlive (fractions falling during clash, loser falls more); out.active (phase not dormant/healed); out.outcome. outcomeLine(raid): deterministic herald sentence, e.g. repelled -> `The banners of {target} hold — the {raiderword} raid is thrown back`, raided -> `{raiderword} torches ring {target}; its {defword} muster in the ash` (reuse RACEWORD-style vocab locally). Pure: no Math.random, no Date.now.

### `src/warrender.js`
RUNG 1 render/projection layer — instanced armies, banners, scorch decals, ruins-props and a night-torch particle pool, all driven off warSim.raidStateAt. Owns its own THREE.Group; every material is a NodeMaterial built ONCE with a TSL colorNode animated only via a shared uTime uniform (S1 law); per-unit/per-mark variation rides on custom instanced float attributes read in the colorNode (aFaction/aFall/aHeal) — the exact civrender.js aRuin/aWarm pattern, proven on the WebGL2 default host. Draw-call budget: 1 warriors InstancedMesh + 1 banners + 1 scorch + 1 props + 1 torch Points = 5 draws regardless of raid count. Distance-culled and refresh-throttled like civrender.

**Contract:** export function createWarRender(planet, warSim, seed) -> { group, update(dt, camera, sunDir?), setNarrator(fn) }. Internals: build ONE low-poly merged, vertex-colored warrior geo (mergeGeometries idiom from caravans.js/civrender.js: torso+head+a spear/shield box) with custom instanced attrs aFaction(0=defender,1=attacker), aFall(0..1), read in a MeshStandardNodeMaterial colorNode = mix(defenderTint,attackerTint,aFaction) darkened by aFall; capacity = RAID_CAP*(ATK_MAX+DEF_MAX). Banners: pole+cloth merged geo, aFaction tint + aHeal fade, capacity RAID_CAP*BANNERS_PER_RAID. Scorch: a flat down-facing disc/quad geo, instanceColor written CPU-side (lerp scorch↔ground tone by healFrac) — OR an aHeal float attr + colorNode mix to WEATHER-ground; hide via degenerate scale when healed. Props: broken-cart geo (reuse caravans buildCartGeometry silhouette, tilted). Torch pool: events.js firework-pool clone (parallel typed arrays, additive Points, TTL fade) — the ONE place Math.random is allowed (ephemeral, never read back). update(dt,camera,sunDir): warClock+=dt (accumulated, the only clock); uTime uniform=warClock; camDist cull beyond VISIBLE_DIST (~3.2R) hides all meshes; throttle a refresh every STATE_REFRESH (~0.4s) OR every frame while any raid is 'marching'/'clashing' (units move); for each raid read raidStateAt, compute each unit dir = offsetPoint(slerp(sourceDir/musterDir/battlefieldDir per phase, ease(t)), slot.bearing, slot.dist), sampleHeight+orientOnSurface+setMatrixAt (degenerate scale to hide inactive slots), set aFall from slot.fallAt vs clashT and side's alive-fraction; write banner/scorch/prop matrices+heal; spawn torch embers at living-unit positions while phase==='clashing' AND night (sunDir·battleUp < 0.1); on the per-raid transition into 'aftermath' (track lastPhase) call the narrator with warSim.outcomeLine(raid) exactly once. setNarrator stores the callback (default no-op). Uses placement.js tangentBasis/orientOnSurface; local offsetPoint/slerp copies per codebase convention.

## main.js wiring
// --- add to the import block (near createCivSim/createCivRender, line ~27) ---
import { createWarSim } from './warsim.js'
import { createWarRender } from './warrender.js'

// --- add a module-scope handle beside `let civRender = null` (line ~149) ---
let warRender = null

// --- inside the EXISTING setTimeout(..., 6500) that builds the civ layer ---
// (main.js already traverses world.group there for anchorsByProject). Extend
// that block to build a richer settlement snapshot and construct the war layer
// from the SAME settled data, so it never races world.js's async population:
setTimeout(() => {
  const anchorsByProject = new Map()
  world.group.traverse((o) => {
    const s = o.userData && o.userData.settlement
    if (s && s.anchorDir) anchorsByProject.set(s.project, s.anchorDir)
  })
  // (existing civ construction stays exactly as-is, using anchorsByProject.values())
  const civSim = createCivSim(planet, SEED, Array.from(anchorsByProject.values()))
  civRender = createCivRender(planet, civSim, SEED)
  scene.add(civRender.group)
  window.__planet.civSim = civSim
  window.__planet.civRender = civRender

  // NEW — build the war layer from a joined snapshot (world.list() rows carry
  // `structures`; the traversal carries anchorDir/groundR/race/structureDirs).
  const recByProject = new Map()
  world.group.traverse((o) => {
    const s = o.userData && o.userData.settlement
    if (s && !recByProject.has(s.project)) recByProject.set(s.project, s)
  })
  const settlementSnapshot = []
  for (const row of world.list()) {
    const s = recByProject.get(row.project)
    if (!s) continue
    settlementSnapshot.push({
      project: row.project, name: row.name, race: row.race, structures: row.structures,
      anchorDir: s.anchorDir, groundR: s.groundR, structureDirs: s.structureDirs || [],
    })
  }
  const warSim = createWarSim(planet, settlementSnapshot, SEED)
  warRender = createWarRender(planet, warSim, SEED)
  warRender.setNarrator((line) => herald.narrate(line)) // route outcome to the DOM chronicle
  scene.add(warRender.group)
  window.__planet.warSim = warSim
  window.__planet.warRender = warRender
}, 6500)

// --- in the render loop (renderer.setAnimationLoop), beside civRender.update ---
// sunDirScratch already exists in main.js; pass the world-space sun dir so torches
// only light at night on the battlefield.
if (warRender) warRender.update(dt, camera, sky.getSunDir(sunDirScratch))

## Determinism
All world state derives from util.js seeds (rngFromString(seed|raid.id + ':tag'), hash01); the sole time source is warClock accumulated from dt in warrender.update, and raidStateAt(raid, warClock) is a pure function of it (idempotent over EPOCH → reload/scrub replays identically). Seeded: raid pairings (isRaider + attractiveness with hash01 tie-break), schedule (epochOffset = idx*EPOCH/RAID_CAP), outcome (hash01(id+':outcome') vs strength ratio), formation slots + casualty fall-times (rngFromString(id+':atk:'+i)/':def:'+i), all mark/battlefield positions (seeded offset rings, clearance-tested). Unit motion is pure interpolation of the clock (offsetPoint over a slerp of source/muster/battlefield dirs), not integrated velocity — no per-frame RNG. warsim.js contains ZERO Math.random/Date.now tokens (grep-clean). warrender.js uses Math.random in exactly one isolated block: the torch/ember Points pool jitter — ephemeral presentation spectacle never read back into sim state, the identical exemption events.js's firework pool and civrender's beacon flicker already ship under. No Date.now anywhere. window.__planet.warSim.raids and every outcome are stable across reloads for a fixed ?seed.

## Covenant
Session structures are immutable history and are touched by exactly ZERO write paths in this epic. Enforcement is layered: (1) OWNERSHIP — warsim.js is pure data (no THREE, Vector3 from the 'three' core like civsim.js) and warrender.js owns its own THREE.Group added alongside (never inside) world.group; neither ever traverses-to-write world, ever reads world.js internals, or ever holds a mutable reference to a session mesh. They only READ a one-time settlement snapshot (project, anchorDir, groundR, race, structures, structureDirs) and planet.sampleHeight/isLand/biomeAt. (2) SPATIAL CLEARANCE (the core guarantee) — warsim computes every battlefield midpoint, muster/defensive line, banner dir, scorch dir and prop dir by rejection sampling against a covenant set = every session anchorDir PLUS every session structureDir, requiring ≥ MARK_CLEARANCE (~0.03 rad) great-circle clearance; an expanding-ring search widens the offset until clear, and if no clear battlefield exists near a target the raid is skipped (warn-once) rather than placed on a record. Muster and defensive lines are offset OUTWARD from the target anchor toward the incoming enemy, so armies form up outside the settlement footprint. A 'raided' outcome scorches the GROUND AROUND the settlement — the buildings are never recolored, moved, scaled, or hidden. (3) HEALING — every mark is a pure function of the war clock, so healing is automatic and total: scorch decals lerp instanceColor scorch→sampled-ground-tone over HEAL_DUR (~20s) then shrink to a degenerate (0,0,0) scale (civrender's hide idiom) and are freed; banners fade and are removed by epoch end (events.js's capped-list removal idiom); sim-owned ruins-props (broken carts, fallen palisades) are removed at epoch end. Because raidStateAt(raid, clock) is idempotent over EPOCH, the world returns bit-for-bit to its pre-raid visual state every cycle. (4) SIM-OWNED PROPS ARE FAIR GAME — raiding parties, defenders, camps, carts, banners, scorch, and ruins-props are all warrender-owned and may be freely spawned/despawned/destroyed. Acceptance asserts this: capture a session structure's world matrix from window.__planet before a raid and after full heal — it must be identical. Rung 3 sieges surround at an offset ring and NEVER enter or overlap the footprint; a lifted siege heals identically.

## Risks
- WebGPU-backend custom instanced-attribute gap: aFaction/aFall/aHeal are custom per-instance vertex attributes read in TSL, which the true-WebGPU backend can fail to compile (the documented gap). Mitigation: the default host is WebGL2 (forceWebGL) where civrender.js already ships this exact pattern; keep marks' color on instanceColor (a three built-in) where possible and hide via degenerate instance-matrix scale (WebGPU-safe); verify on ?renderer=webgpu and warn-once if a material degrades — accepted risk identical to civrender's.
- Covenant clearance on dense/clustered worlds: a battlefield or scorch ring may find no spot ≥ MARK_CLEARANCE from every anchor+structureDir. Mitigation: expanding-ring search that widens the offset, then SKIP the raid (warn-once) rather than ever place a mark on a record — the covenant wins over spectacle.
- Async settlement race: snapshotting before world.js finishes its /api/sessions poll would miss settlements or raiders. Mitigation: build the war layer inside main.js's existing 6500ms setTimeout (the same settled window civ uses); if <2 settlements or no raider/target pair exists, build nothing and warn once.
- Per-frame sampleHeight for many units: 2 concurrent raids × up to ~24 units each = ~<100 sampleHeight+matrix writes per near-frame. Bounded by camera cull, small counts, RAID_CAP=4 and ≤2 overlap; matches caravans.js's already-accepted per-frame cost.
- Night detection depends on sunDir: if main.js does not pass sky.getSunDir, torches must degrade to off, never crash (sunDir optional in update).
- Torch Math.random must never leak into world state — keep the pool's arrays isolated; unit/mark/outcome/schedule positions never read particle state.
- Rung 2 conflict signal reliability: gh PR CONFLICTING status needs auth/network; mirror gitinfo.js's existing warn-once-and-skip degradation so a missing signal simply yields no data-driven skirmishes.
- No prosperity field exists on session settlements (confirmed) — rung 1 uses `structures` count as the strength/attractiveness proxy; rung 2 upgrades it with polled commit/activity tallies. Documented so no builder hunts for a nonexistent field.

## Acceptance
- RUNG 1 — theater plays: with a seed that yields ≥2 settlements including a tmp/scratch or orc raider and a richer target, ~7s after load a party of instanced warriors marches from the raider settlement toward the target, STOPS outside its footprint at a muster point, defenders emerge and form a line, and the two formations clash at an on-land midpoint.
- RUNG 1 — outcome surfaces: at the clash's end the Aemunis Herald strip shows a 'raid repelled' or 'settlement raided' line exactly once per raid; winner-colored banners appear at the battlefield and scorch decals appear on the GROUND around the site.
- RUNG 1 — covenant heal (the critical check): capture a session structure's world matrix from window.__planet.world before a raid; after the ~20s heal window the scorch fully fades+vanishes, banners are removed, and that structure's matrix is byte-identical — nothing was moved, recolored, scaled, or hidden.
- RUNG 1 — night torches: drag the sun below the horizon (or wait for night); warm torch embers glow among the clashing units and are absent by day.
- RUNG 1 — determinism: reload with the same ?seed — identical raids, pairings, outcomes and timings; window.__planet.warSim.raids is stable and equal across reloads.
- RUNG 1 — budget: renderer.info.render.calls rises by ≤ ~5 when a raid is on-screen; fps stays ≥55; zooming past the cull distance hides the army meshes while the sim keeps advancing (units are in the correct place on return).
- RUNG 2 — a merge conflict between two related projects (or an injected /api/events {kind:'conflict'}) triggers a border skirmish between their settlements within a poll cycle, narrated by the herald; territory shading tints terrain between rival clusters and heals (no permanent stain); a caravan/airship supply cart is intercepted on its route (scorch on the road + scattered supply props) and the route resumes; raid targeting visibly favors the most prosperous (most structures/active) settlement.
- RUNG 3 — repeated conflicts lower a faction standing and spawn more raids; a pr-merge between rivals raises standing to a treaty (raids cease, peace banner/road, herald 'peace'); a siege surrounds a settlement at an offset ring that never overlaps any building, leaves a scorch ring, and heals completely when lifted; unit counts scale with historical session/structure counts; asserting a besieged structure's matrix is unchanged across the whole siege passes.
