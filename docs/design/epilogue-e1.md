# E1 — visible rivers + terrain relief from erosion, on a committed CPU-baked heightfield

E1 replaces planet.js's per-call analytic sampleHeight (5+ simplex-fbm stacks) with a committed, eroded lat/long heightfield: one Float32/Uint16 equirect grid per seed that is the SINGLE SOURCE OF TRUTH for both the terrain mesh displacement AND an allocation-free bilinear sampleHeight(dir) of identical signature — so all 30+ callers keep working unchanged (and get faster). Deterministic CPU hydraulic erosion carves broad drainage valleys into that grid (visible relief), and a drainage/flow network traced from the same grid drives thin water-ribbon rivers that follow the valleys and tuck under the ocean shell at their mouths. The eroded grid + river polylines are baked once by a Node script and committed as public/bakes/<seed>.hf.bin (loaded at runtime, never recomputed — protecting determinism even if erosion ever moves to non-bit-identical GPU compute); uncommitted custom seeds fall back to the same deterministic CPU bake. Rivers render as one static draw call on both WebGL2 and true-WebGPU, animated only by a uTime uniform.

## New files

### `src/terrainField.js`
THREE-free extraction of planet.js's analytic terrain math so BOTH the browser (planet.js) and the Node bake script can build a byte-identical height function. Pure ESM, no 'three/webgpu' import, so Node can import it. This is the seam that lets erosion be baked offline and loaded at runtime.

**Contract:** export function createTerrainField(seed) -> { analyticHeight(dir)->number, continentField(x,y,z)->number, beltField(x,y,z)->number, moistureAt(dir)->0..1, capThresholdAt(dir)->number }. analyticHeight is the CURRENT sampleHeight body moved VERBATIM (same constants CONTINENT_SCALE..HEIGHT_MAX, same fbm/ridged/smoothstep calls, same clamp(h,HEIGHT_MIN,HEIGHT_MAX)) — a no-op refactor: output must be bit-identical to today's planet.js sampleHeight for every dir. Noise fields (nContinent,nBelt,nMountain,nDetail,nOceanFloor,nMoisture,nJitter,nCap,nWarpX/Y/Z,nIsland) built here from makeNoise3D(seed+':...'). Imports only from './util.js'. dir is any {x,y,z} (need not be THREE.Vector3).

### `src/heightfield.js`
The baked equirect grid container: allocation-free bilinear samplers (the sampleHeight REPLACEMENT), equirect index math shared with the bake, and (de)serialization of the committed .bin. THREE-free so Node can encode and the browser can decode.

**Contract:** export function makeHeightField({width,height,hMin,hMax,heights:Uint16Array,flow?:Uint16Array,rivers:Array}) -> hf. hf.sampleHeight(dir)->number: normalize dir, theta=acos(clamp(y/r,-1,1)) in [0,PI], phi=atan2(z,x) in [-PI,PI]; u=phi/(2PI)+0.5 (WRAP), v=theta/PI (CLAMP rows, no v-wrap — poles); bilinear over 4 texels, dequant deq(u16)=hMin+(u16/65535)*(hMax-hMin); ZERO allocation, ZERO THREE. hf.sampleFlow(dir)->0..1 (same math over flow grid if present, else 0). hf.getRiverNetwork()->{paths:[{nodes:Float32Array[x,y,z per node], widths:Float32Array, order:int, mouthUnder:bool}]}. hf.width/height/hMin/hMax exposed. hf.hash()->number (sum of quantized heights mod 2^32 + river node count) for determinism checks. Also: export function encodeHeightField(hf)->ArrayBuffer and export function decodeHeightField(arrayBuffer)->hf. Binary layout (LE): magic 'SKHF'(4B), version u32=1, width u32, height u32, hMin f32, hMax f32, riverCount u32, then heights u16[W*H], then per-river {u32 nodeCount, nodeCount*(f32 x,y,z,width), u8 order}. Imports clamp from './util.js' only.

### `src/erosion.js`
Deterministic CPU hydraulic+thermal erosion and drainage extraction over an analytic height function. Produces the eroded grid (broad valleys = visible relief) and the traced river polylines. Pure — the ONE place erosion is computed; its committed output is what ships. No THREE, no Math.random/Date.now.

**Contract:** export function bakeErosion({seed, analyticHeight, width=2048, height=1024}) -> heightField (a makeHeightField result with eroded heights + flow + rivers). Pipeline (all deterministic): (1) FILL raw grid: for each texel center compute dir via inverse-equirect, h=analyticHeight(dir); track hMin/hMax. (2) PRIORITY-FLOOD depression fill (Barnes 2014): min-heap keyed by filled height, tie-break by linear index (NO rng); ocean cells (h<=SEA_LEVEL) and grid poles are outlets; guarantees every land cell drains to sea. (3) D8 flow-accumulation on the filled grid, columns WRAP, rows clamp; process cells in decreasing filled height (topological), accum starts 1, add downstream. fNorm=log(1+accum)/log(1+maxAccum). (4) CARVE broad valleys: for land cells lower height by CARVE_MAX*pow(fNorm,CARVE_GAMMA)*highlandBoost(h)*coastTaper(h), then SPREAD the carve over a ~6-8 texel radius (separable Gaussian on the carve field) so valleys exceed the mesh Nyquist (~6 grid texels) and render as relief, not sub-mesh spikes. coastTaper->~0 below SEA_LEVEL+0.006 so buildable lowlands (placement's MAX_BUILD_HEIGHT band) barely move; highlandBoost raises carve in mountains. Never carve a land cell below SEA_LEVEL except within RIVER_THRESHOLD channels reaching the coast (rivers meet the sea). (5) THERMAL relax 1-2 passes (talus slope limit) to kill single-texel artifacts. (6) RIVER extraction: cells with accum>RIVER_THRESHOLD are river cells; from each headwater (river cell with no upstream river contributor) trace the D8 drainage tree downstream to ocean, emitting one polyline per stream segment between confluences; Strahler order -> width tier; width also from sqrt(fNorm); Chaikin-smooth each polyline for a clean ribbon path; mark the mouth node mouthUnder. Constants exported for ART tuning: CARVE_MAX~0.010, CARVE_GAMMA~0.6, RIVER_THRESHOLD, CARVE_SPREAD_TEXELS~7, THERMAL_TALUS. Any tie-break/jitter uses rngFromString(seed+':erosion'). Imports SEA_LEVEL, clamp, rngFromString, hash01 from './util.js'.

### `src/rivers.js`
The main.js-wired render module: turns planet.getRiverNetwork() into thin draped water ribbons that sit in the carved valleys and tuck under the ocean at coasts. One static geometry, one draw call, both backends.

**Contract:** export function createRivers(planet, seed) -> { group:THREE.Group, update(dt) }. Reads planet.getRiverNetwork() at construction (network is ready the moment createPlanet resolves — NO settlement wait like roads.js). Bakes ONE non-indexed BufferGeometry: per polyline, drape a two-vertex-wide strip (mirror roads.js bakeEdge) at radius planet.sampleHeight(nodeDir)+RIVER_LIFT (RIVER_LIFT~0.0005 above the carved valley floor), half-width from per-node width tapering headwater->mouth; per-vertex attribute aRiverT (0 head ->1 mouth) and aFlow. Compose ALL transforms from positionLocal (NOT positionGeometry/custom instanced attrs — WebGPU-safe). Material: MeshStandardNodeMaterial, transparent, depthWrite:true, roughness~0.5, colorNode = ocean.js-style depth/fresnel mix of uStopShallow(0x8fe2d1)->uStopMid(0x2f8fa8)->uStopDeep(0x0f3a66) with a downstream shimmer = sin(aRiverT*K + uTime) blended <=1.0 — NEVER >1.0 headroom (ART.md §2.5: water must not bloom). Node graph built ONCE; update(dt){ uTime.value += dt } is the only animation (no allocation). Terminate polylines one cell past the coast so ribbon ends sit just below SEA_LEVEL and fade under the translucent ocean shell. Expose group.userData.riverHash = planet heightfield hash for verify. Imports THREE from 'three/webgpu', TSL nodes, SEA_LEVEL/hash01/clamp from util, tangentBasis from placement.js.

## Determinism
SOURCE OF TRUTH: the committed public/bakes/<seed>.hf.bin, loaded via fetch and NEVER recomputed at runtime for shipped seeds — the eroded shape and rivers are byte-identical on every machine/reload regardless of GPU (the audit's mandate satisfied because runtime does zero erosion compute for shipped seeds). Uncommitted custom ?seed= values fall back to bakeErosion() on CPU: pure IEEE-754 JS float math over an integer grid (priority-flood heap tie-broken by linear index, D8 flow accumulation, separable Gaussian carve, thermal relax) — bit-identical across machines and identical to what the bake script commits. SEEDING: noise fields via makeNoise3D(seed+':tag') (util.js, unchanged); the erosion core needs no RNG, and any river-width micro-jitter uses hash01(seed+':river:'+i) / rngFromString(seed+':erosion') — never Math.random. NO Math.random or Date.now in terrainField.js/heightfield.js/erosion.js/rivers.js/bake-heightfield.mjs (grep-enforced acceptance check). The ONLY runtime time source is rivers.js update(dt) accumulating dt into a uTime uniform for shimmer — presentation-only, never feeds world state (same discipline as planet.js's ocean uTime). Placement stays deterministic (same code, deterministic grid). Determinism hash heightfield.hash() (sum of quantized heights mod 2^32 + river node count) exposed on window.__planet.planet.bakeHash and window.__planet.rivers.group.userData.riverHash for the verify sweep.

## Covenant
E1 is a terrain feature, not a simulation, but it respects the immutable-history spirit. (1) NOTHING in E1 destroys, moves, or overwrites a session structure: rivers and carved valleys are ADDITIVE terrain geometry; world.js structures are never touched, read, or referenced by terrainField/heightfield/erosion/rivers. (2) The one behavioral consequence is that changing the terrain shape re-lands settlement placement deterministically — this is a controlled ONE-TIME terrain-version bump (the committed bake pins it), explicitly the class of change ART.md §3.5 already sanctions ('settlement layout will reshuffle at coasts ... accepted, pre-1.0 world'), NOT a live/per-launch disturbance. (3) Placement is protected where it matters: the erosion carve tapers to ~zero in the buildable lowland band (below SEA_LEVEL+0.006, the placement MAX_BUILD_HEIGHT zone) and only thin high-drainage channels drop, so the broad ground settlements occupy is left intact — settlements move minimally, if at all. (4) Rivers are static (no heal cycle needed): they are baked once and only shimmer via a presentation uTime; they never advance, flood, or reshape around structures, so there is no marking-and-healing obligation. If a future sim feature (E-SIM raids/floods) ever interacts with rivers, THAT feature carries the additive/heal covenant — E1 just provides immutable geography.

## Risks
- createPlanet becomes async (adds `await` at main.js:59). Contained: main.js is the ONLY caller (grep-confirmed: no other src/ or tests/ reference) and is already an async module. Risk LOW; the returned object is a strict superset so all 30+ downstream consumers are untouched.
- Erosion carves the terrain -> settlement layout shifts ONCE for the default seed. This is the same class of change ART.md §3.5 already ACCEPTS ('Settlement layout will reshuffle at coasts ... accepted, this is still a pre-1.0 world'). Mitigated so the shift is minimal: carve tapers to ~0 in the buildable lowland band (below SEA_LEVEL+0.006, i.e. placement's MAX_BUILD_HEIGHT zone) and only high-flow channel cells drop — broad lowlands where settlements sit are untouched. The committed bake makes the shift a one-time deterministic terrain-version bump, not a per-launch wobble.
- Committed .bin size (~4MB/seed heights + river nodes) in git. Mitigated by Uint16 quantization over the narrow [0.975,1.06] height range (1.3e-6 precision, far below any visible band) and gzip-friendly smooth data; commit ONLY shipped seeds; add public/bakes/*.bin to .gitattributes as binary. If size bites, store heights-only and re-derive flow+rivers deterministically at load (flow accumulation is fast, bit-identical CPU) — noted as a fallback, not the default.
- Mesh (~163k verts, ~0.5deg spacing) is ~3x COARSER than the 2048x1024 grid (~0.176deg). A 1-texel carved channel would alias below the mesh and NOT show as relief. Mitigated by design: erosion carves BROAD valleys (CARVE_SPREAD_TEXELS~7 Gaussian widening + thermal relax) that exceed the mesh Nyquist (~6 grid texels) and read as real relief, while the THIN water line is the rivers.js ribbon's own fine geometry sitting in the valley floor — the correct real-world read (broad valley, thin river). If valleys still read too soft, raising IcosahedronGeometry detail 128->160 is the escape hatch (flagged, MEASURE fps/mem first — it touches the whole terrain).
- Bilinear grid sampleHeight differs slightly from the old analytic values near coasts, which can flip a thin fringe of isLand cells (threshold SEA_LEVEL+0.0015). Deterministic and bounded; same class as the §3.5 coast reshuffle. Because the MESH and the SAMPLER now read the SAME grid, props/camera-floor/roads still hug the visible surface exactly (arguably tighter than today) — no floating or sunk props.
- THE COVENANT: E1 is terrain, not simulation, but it must not violate the 'structures are immutable history' spirit. Rivers and carve are ADDITIVE terrain that never destroy/move/overwrite a session structure; a river can only pass NEAR or under a static ribbon of water. Residual risk: a river ribbon could visually cross a building footprint. Mitigated because carve+rivers taper out of the buildable lowland band; if a mouth passes a coastal settlement it reads as a harbor river, thematically fine. Flagged for the integration eye check.
- Equirect pole singularity: bilinear degenerates as v->0/1. Poles are ice caps (no rivers, no settlements). Sampler clamps v rows (no v-wrap) and accepts pole distortion under the ice. Risk LOW.
- true-WebGPU backend WGSL gaps (pointUV, custom per-instance vertex attributes). Avoided: rivers.js uses NO point sprites and composes transforms from positionLocal, not custom instanced attributes; the two per-vertex attributes (aRiverT, aFlow) are plain geometry attributes on a normal Mesh (like roads.js color / ocean.js aDepth), which compile on both backends. Verified by the both-backends acceptance check.

### BT3 — Deterministic CPU erosion + drainage extraction
files: src/erosion.js

Create src/erosion.js, THREE-free (import SEA_LEVEL,clamp,rngFromString,hash01 from './util.js', and makeHeightField from './heightfield.js'). Export `function bakeErosion({seed, analyticHeight, width=2048, height=1024})` returning a makeHeightField (eroded heights as Uint16, flow as Uint16, rivers array). Pipeline, ALL deterministic (no Math.random/Date.now): (1) Allocate Float32 raw[W*H]. For each row r (v=(r+0.5)/H mapped to theta=v*PI... use texel-center convention consistent with heightfield.js sampleHeight inverse: for col c, u=(c+0.5)/W, phi=(u-0.5)*2PI, theta=v*PI where v=r/(H-1); dir.x=sin(theta)cos(phi), dir.y=cos(theta), dir.z=sin(theta)sin(phi)), raw[i]=analyticHeight(dir); track hMin,hMax. (2) Priority-flood depression fill: min-heap keyed by filled height, tie-break by index; push all ocean cells (raw<=SEA_LEVEL) and the two pole rows as outlets with filled=raw; pop lowest, for each 8-neighbor (columns WRAP, rows clamp) not yet finalized set filled=max(rawNeighbor, current+1e-6), push. (4) Carve: carveField[i]=CARVE_MAX*pow(fNorm[i],CARVE_GAMMA)*highlandBoost(raw[i])*coastTaper(raw[i]) where coastTaper=smoothstep(SEA_LEVEL+0.006, SEA_LEVEL+0.02, raw) (0 in buildable lowlands), highlandBoost=0.4+0.6*smoothstep(SEA_LEVEL+0.01,HEIGHT peak,raw).
