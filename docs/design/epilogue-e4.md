# E4 — Living-World Remainder: Ruins, Seasons, Fish Schools, Ambient Sound, Poster Export, Auto-Tour

Six independent, self-contained modules that finish the living world. Three render into the scene as instanced/uniform-animated groups (RUINS ancient props, SEASONS snow-line shell + read API, FISH SCHOOLS boids), and three are non-scene client features wired through the existing UI patterns (AMBIENT SOUND WebAudio + mute, POSTER EXPORT high-res PNG, AUTO-TOUR cinematic camera path). Every module follows the createX(...seed) -> { group?/mesh?, update(dt) } contract, is seeded via src/util.js (no Math.random/Date.now in state), and builds each TSL node graph once with uniform-only animation. Nothing touches session structures: ruins are pre-existing ancient props, the snow shell is additive over terrain and recedes every cycle, and the sim-owned fish/tour/audio never write history.

## New files

### `src/ruins.js`
Decayed ancient-ruin props (crumbled wall segments, a toppled column, an overgrown rubble mound) scattered at seeded ancient sites that are independent of — and never overlap/destroy — session settlements. One instanced draw call from a merged low-poly ruin kit; static and additive (covenant).

**Contract:** export function createRuins(planet, seed) -> { group: THREE.Group, update(dt) }. Builds ONE merged, vertex-colored ruin geometry (broken-wall boxes + a snapped column cylinder + a low rubble icosahedron), painted with paintFlatColor-style weathered greys/mossy tints, via mergeGeometries with a warn-once non-indexed fallback (mirror flora.js buildTreeGeometry). One THREE.InstancedMesh(mergedGeo, MeshStandardNodeMaterial{vertexColors,flatShading,roughness:1}) sized RUIN_SITE_MAX * PIECES_PER_SITE (~12 sites x ~6 pieces). Placement: seeded rejection loop over randUnit3(rng)+planet.isLand+planet.biomeAt (reject slope>~0.45, polar>~0.4, landT outside ~0.05..0.6) with a createSpacingGrid-style min-angular-spacing between SITES (~0.05 rad) so ruins never clump; each accepted site scatters PIECES pieces on small tangent offsets, each planted with a plantedMatrix-style basis (local +Y = surface normal from planet.sampleHeight(dir), seeded yaw/tilt/scale). All transforms baked into instanceMatrix at build (NO custom per-instance vertex attributes — WebGPU-backend safe). update(dt): only mesh.visible = camera.position.length() < PROP_VISIBLE_DIST(~3.2); no per-frame matrix work. No animated uniforms.

### `src/seasons.js`
The authoritative global season clock (spring→summer→autumn→winter) plus a self-rendered, terrain-hugging snow-cover shell whose snow line descends in winter and recedes to peaks/poles in summer via ONE animated uniform. Exposes getters other modules read for foliage tint, sea-ice extent and daylight feel.

**Contract:** export function createSeasons(seed) -> { group, update(dt), getPhase01(), getSeasonIndex(), getSeasonName(), getFoliageTint(outColor), getDaylightFactor(), getSeaIceThresholdBias(), uSnowLine, uFoliageTint }. group holds the snow shell: a THREE.IcosahedronGeometry(1, ~6) whose every vertex is displaced along its direction to planet.sampleHeight(dir)+SNOW_LIFT(~0.001), with a per-vertex Float32 attribute 'snowPotential' baked from latitude(|dir.y|) + altitude(landT) + seeded jitter (rngFromString(seed+':seasons:snow')). Material = MeshStandardNodeMaterial{ colorNode = snow-white; the graph is built ONCE }; visibility gated by comparing attribute('snowPotential','float') against a uniform uSnowLine (opacity/alphaTest style: snowPotential above uSnowLine → opaque, soft edge via smoothstep). update(dt): advance a phase clock (SEASON_PERIOD seconds = one full year, ~240s) and WRITE uSnowLine.value (winter low → snow reaches lower latitudes/altitudes; summer high → only peaks), uFoliageTint.value (green→gold→brown→pale seasonal THREE.Color), plus cache scalar daylightFactor(~0.82 winter..1.08 summer) and seaIceThresholdBias(negative in winter = more ice). getFoliageTint copies the cached color into outColor. No Math.random; only uniform.value writes animate — no graph edits.

### `src/fishschools.js`
Boids fish schools shimmering just under the ocean surface, tethered to seeded shallow-water anchors so they stay visible in the shallows. One InstancedMesh (one draw call), camera-culled.

**Contract:** export function createFishSchools(planet, seed) -> { group, update(dt, camera) }. ONE merged low-poly fish geometry (a flattened diamond body + a small tail triangle, DoubleSide so the flat blades never cull — birds.js/sealife.js idiom), vertex-colored a muted countershaded tone (ART.md: no glow/bloom-crossing). ONE THREE.InstancedMesh sized SCHOOL_MAX * FISH_PER_SCHOOL (~6 schools x ~28 = ~170). Anchors: seeded search reusing a nearCoast-style ring probe (planet.isLand on a ~0.02 rad ring) for shallow coastal water. Each fish gets a seeded initial tangent-plane offset + heading (rngFromString(seed+':fish:school:i')). update(dt,camera): camera-cull beyond ~2.2R (skip all matrix work, keep state ticking cheaply); when near, run CPU boids per school — separation + alignment + cohesion + a spring tether to the school anchor + a hard constraint keeping each fish between SEA_LEVEL-0.006 and SEA_LEVEL-0.001 — integrate with dt, then compose each instanceMatrix (position = fishDir*(SEA_LEVEL+depth), up = surface normal at fishDir, forward = normalized velocity; tangentBasis from placement.js). ALL animation is CPU→instanceMatrix; NO custom per-instance vertex attributes (WebGPU-backend safe). frustumCulled=false on the mesh (fish drift).

### `src/ambient.js`
Procedural WebAudio ambience: layered wind (filtered noise), ocean surge (slower filtered noise), and a faint activity bed scaled by active-agent count. Starts only on a user gesture; a mute toggle drives it. No scene object.

**Contract:** export function createAmbientSound(seed) -> { start(), toggleMute(), setMuted(on), isMuted(), update(dt, camera, world) }. Returns NO group (non-scene, herald.js-style). Internals: a lazily-created AudioContext (constructed AND resumed inside start(), which the UI mute button calls from within its click handler = the required user gesture — nothing sounds before that, autoplay-policy safe). A seeded white-noise AudioBuffer filled from an rngFromString(seed+':ambient:noise') loop (NOT Math.random), looped through two BiquadFilter paths (a breathy high-passed 'wind' layer and a low-passed rumbling 'ocean' layer) into per-layer GainNodes and a master gain. update(dt,camera,world): modulate wind gain up with camera altitude (camera.position.length()), ocean/surf gain up near the surface, and a faint 'activity' gain from world.stats.agents — smoothed toward targets (no snap). Mute persists to localStorage (guarded). setMuted ramps the master gain (no clicks). Safe no-op if WebAudio is unavailable.

### `src/poster.js`
High-resolution poster capture: re-renders the current camera view at 2–4× through the EXISTING post-processing chain and downloads a PNG. No scene object.

**Contract:** export function createPosterExport({ renderer, post, camera }) -> { capture(scale = 3) }. capture() returns a Promise: snapshot current renderer size + pixelRatio + camera aspect; add a body.capturing class (CSS hides #planet-ui + .hud for a clean plate); renderer.setSize(innerWidth*scale, innerHeight*scale, false) with updateStyle=false so the on-page canvas CSS size never jumps (aspect is unchanged, so camera.updateProjectionMatrix() is enough); await post.renderAsync() to draw a fresh full-res frame through scattering→clouds→bloom→dither; read the pixels via renderer.domElement.toDataURL('image/png') (WebGL2 default backend, canvas holds final sRGB pixels after the awaited render) with a toBlob fallback; trigger an <a download="<planetName>-poster.png"> click; then RESTORE size/pixelRatio/aspect + updateProjectionMatrix and remove body.capturing. Runs independently of the animation loop (awaits its own render). Note the true-WebGPU backend (?renderer=webgpu) may need the toBlob path; document it.

### `src/autotour.js`
A seeded cinematic camera tour that flies between notable settlements via cameraFeel, dwelling with a gentle auto-rotate at each stop, and cooperating cleanly with OrbitControls (any user input stops the tour). No scene object.

**Contract:** export function createAutoTour(planet, world, cameraFeel, controls, camera, seed) -> { start(), stop(), toggle(), isRunning(), update(dt) }. Returns NO group. On start(): read world.getAnchors() (settlement summaries incl. live read-only anchorDir + structure count), weight 'notable' by structure count, build a deterministic visiting order via a seeded Fisher–Yates shuffle (rngFromString(seed+':autotour')), save controls.autoRotate, and begin. State machine per stop: FLY (cameraFeel.flyTo(anchorDir, arriveDist=max(1.3,0.45*camera.position.length()))) → wait until cameraFeel.isFlying()===false → DWELL (DWELL_SECONDS ~6s with controls.autoRotate=true, controls.autoRotateSpeed low) → advance to next (wrap around). update(dt): tick the dwell timer and kick the next flyTo when the current phase completes. Cooperation: a pointerdown listener on controls.domElement calls stop() (cameraFeel already cancels its own flight on pointerdown); stop() restores controls.autoRotate to the saved value. If world.getAnchors() is empty, start() is a no-op. Presentation-only; deterministic order from the seed.

## Determinism
Every structural random choice in all six modules derives from rngFromString(SEED + ':tag') / hash01 / makeNoise3D from src/util.js — never Math.random, never Date.now. RUINS: site directions, per-site piece counts, yaw/tilt/scale, overgrowth tint all pulled from rngFromString(seed+':ruins:...') streams; placement is a seeded rejection loop (same accept/reject sequence every launch), so the ruin field is a pure function of the seed. SEASONS: the season phase is an accumulated presentation dt clock starting at phase 0 (deterministic), snow-shell per-vertex jitter from rngFromString(seed+':seasons:snow'); no other randomness. FISH SCHOOLS: coastal anchors, per-school member counts, and each fish's initial tangent-plane offset + heading come from rngFromString(seed+':fish:...'); the boids integration thereafter is driven ONLY by the presentation dt clock (allowed presentation state, exactly like sealife.js's whale/dolphin pose state and world.js's FX pools) — no Math.random in the flock loop. AMBIENT SOUND: the noise buffer is filled from an rngFromString(seed+':ambient:noise') loop (NOT Math.random), and all gain modulation reads camera altitude + world.stats.agents — no clock. POSTER EXPORT: no randomness. AUTO-TOUR: the stop ordering is a seeded Fisher–Yates shuffle over world.getAnchors() using rngFromString(seed+':autotour'); dwell timing is a presentation dt clock. The only Date.now in the touched files stays where it already is (world.js session lastActive / main.js events polling — session data, not world state); no new module reads a wall clock.

## Covenant
All six respect THE COVENANT. RUINS are ancient, pre-existing, purely additive props seeded independently of sessions; they are static (no update-loop mutation beyond a camera-distance visibility toggle) and can never destroy, move, or overwrite a session structure — they only ever appear AROUND the record at seeded ancient sites. SEASONS' snow-cover shell is an additive terrain-hugging layer that never edits planet.js geometry or any structure; its snow line advances in winter and fully RECEDES (heals) every cycle via a single uniform, leaving no permanent mark. FISH SCHOOLS, AMBIENT SOUND, AUTO-TOUR and POSTER EXPORT are presentation-only (sim-owned props / camera / audio / a screenshot) and write no world state at all. The optional seasons hooks into seaice/flora/sky only modulate their own already-existing uniforms/light values and are fully reversible.

## Risks
- Snow shell overdraw / z-fighting: the seasons snow shell hugs terrain at sampleHeight+SNOW_LIFT and can z-fight the planet mesh or the flora contact-blobs. Mitigate with a small SNOW_LIFT (~0.001), polygonOffset, and depthWrite tuning; keep icosphere detail moderate (~6) so it stays one cheap draw call. It must fully recede in summer (uSnowLine high) so it never becomes a permanent white cap.
- Fish boids cost: CPU boids over ~170 instances every near-camera frame is O(n^2) per school if naive. Keep schools small (~28) and only flock within a school (not across schools), and hard-cull beyond ~2.2R. If it hitches, spatial-hash within a school or reduce FISH_PER_SCHOOL.
- Poster readback on the true-WebGPU backend (?renderer=webgpu): toDataURL may return blank without a preserved drawing buffer. Default backend is WebGL2 (forceWebGL), where toDataURL after an awaited renderAsync works; document the toBlob fallback and that WebGPU-backend capture is best-effort. Also guard against very large scale values (4x on a 4K display) exhausting GPU memory — clamp scale and restore size in a finally block even if render throws.
- Auto-tour vs OrbitControls/photo-mode/cameraFeel contention: cameraFeel already cancels its flight on pointerdown, but the tour state machine must ALSO stop() on that pointerdown or it will immediately re-issue a flyTo and fight the user. autoRotate is shared with setPhotoMode's hook — save/restore it and don't leave it on after stop().
- Ambient autoplay policy: the AudioContext must be created/resumed strictly inside the mute button's click handler; creating it at module load will be suspended by the browser and may log warnings. Verify nothing sounds until the first click.
- Seasons cross-module coupling: flora colorNode edit replaces the default vertexColors path with an explicit attribute('color').mul(uFoliageTint) node — verify canopy AND trunk still read their baked vertex colors (the whole tree geo shares one 'color' attribute) and that flatShading/roughness are unaffected. All three consumer edits (seaice/flora/sky) are guarded so seasons can ship before them.
- main.js ordering: poster needs `post`, which is defined ~line 209 (late). Creating poster earlier throws. autoTour needs world+cameraFeel+controls. Keep the create-site placement exactly as specified.
- Ruins overlapping a session settlement: ruins are seeded independently and never destroy anything (covenant intact), but a ruin could visually sit inside a town. If undesirable, gate ruins behind the same deferred settlement-anchor snapshot main.js already uses for civSim (~line 150) and reject sites near anchors — noted as an optional refinement, not required for covenant compliance.
