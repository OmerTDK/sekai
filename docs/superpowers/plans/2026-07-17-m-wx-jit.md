## Contracts (pinned)

- B1 `createBirds(planet, seed) → { group, update(dt, camera) }` (replaces
  current 2-triangle module; same factory name, new planet param)
- B2 `createSeaIce(planet, seed) → { group, update(dt) }`
- B3 `createWeather(planet, sky, seed) → { group, update(dt, camera) }`;
  sky accessor: `sampleCloudCover(dir:Vector3) → 0..1` CPU-side from the
  already-generated coverage data — no GPU readback, no other sky.js edits
- B4 `createFloods(planet, storms, seed) → { group, update(dt) }`
  (storms API: `getPrimary(outVec3) → strength 0..1`, fills center dir)
- B5 `createTrails(planet, world, seed) → { group, update(dt) }`
  (world API: `forEachWalker(fn(id, dirVec3, race))` — dir is live/read-only)
- B6 `createSeaLife(planet, seed) → { group, update(dt, camera) }`

Planet API: `sampleHeight(dir)→radius`, `isLand(dir)→bool`,
`biomeAt(dir, out)→{h, landT, moisture, slope, polar}` — all synchronous,
allocation-free. `SEA_LEVEL = 1.0` (util.js), `HEIGHT_MAX = 1.06`.

## Height/shell budget (do not trespass)

Surface effects (ice, flood, trails) live within +0.002 of their surface.
Birds: gulls terrain+0.004–0.008, migratory band 1.065–1.073 (shared with
dragon band — sparse, non-colliding by longitude offset). Clouds 1.075+.

## Laws (every builder)

- Deterministic: seed everything via `rngFromString`/`hash01`/`makeNoise3D`
  from `src/util.js`; `Math.random`/`Date.now` banned in world-state code.
- No per-frame allocations in `update`; preallocated pools/scratch only.
- Draw calls: ≤3 per module (instanced/batched/merged).
- Silent-fallback rule: every degradation `console.warn`s exactly once.
- three.js 0.185.1 pinned; onBeforeCompile allowed (WebGL2 era — flagged
  for the M3 TSL port list if used; note it in your report).
- ART.md: muted saturation ceilings, readable silhouettes at distance,
  no neon, matte snow/ice (S5 verdict: stylized, not photoreal PBR).
- `npx prettier --check .` + `npx eslint .` + `node --check <file>` +
  `npm test` green before committing. Commit ONLY owned files,
  conventional message `feat(<module>): …`, no AI attribution.
